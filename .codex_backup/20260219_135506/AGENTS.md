Regole operative (valgono sempre):
- Obiettivo: migliorare qualità (esperienza d’uso, interfaccia, contenuti, collaudo).
- Metodo: osservare → ipotizzare → intervenire → verificare → registrare.
- Ogni modifica deve avere una verifica (build, test, o controllo ripetibile).
- Se serve lavorare “a squadra”, il Coordinatore decide e chiama gli altri agenti.
