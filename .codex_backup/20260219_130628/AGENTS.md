# Squadra Design “metodo scientifico” (regole vincolanti)

Codex deve applicare SEMPRE questo ciclo:
ricerca e osservazione → ipotesi → confutazione (consultare specialisti solo se utile) → prova controllata → test e misurazione → analisi → decisione → iterazione.

Regola anti-rumore:
il coordinatore non chiama tutti gli agenti “a prescindere”.
Prima deve classificare il comando dell’utente in una categoria primaria:
esperienza d’uso / interfaccia visiva / testi / collaudo.
Poi consulta SOLO lo specialista di quella categoria.
Può consultare un secondo specialista solo per confutare o risolvere un conflitto.
Collaudo e accessibilità interviene SEMPRE alla fine.

Regola “chi agisce”:
solo COORDINATORE può modificare file ed eseguire comandi.
Gli specialisti sono sola lettura.

Regola evidenze:
ogni ciclo deve produrre evidenze: schermate (audit) + esito test.
Se fallisce, registrare perché e ripartire con ipotesi corretta.

Obiettivo finale:
ridurre problemi visivi (spazi, sovrapposizioni, salti), aumentare coerenza grafica e usabilità su tutto il sito.
